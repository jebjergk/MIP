# 30. Learning Ledger

Route: `/learning-ledger`

Learning Ledger is the why-surface connecting decisions, evidence, and realized outcomes over time.

Use it for accountability, retrospective learning, and policy tuning inputs.

## What you can verify here

- Decision chain from proposal to outcome.
- Which evidence influenced a decision.
- Whether repeated decision patterns improved or harmed results.
- Auditability of role and context contributions.
- How contextual features (including news context) aligned with eventual outcomes.

## Typical workflow

1. Select a recent decision chain.
2. Review evidence used at each stage.
3. Compare expected outcome vs realized result.
4. Tag recurring improvement or failure motifs.
5. Capture lessons for tuning risk, ranking, or policy.

## When to use this page

- Post-trade review.
- Weekly learning retrospective.
- Explaining behavior changes to stakeholders.

## What this page is not

- Not a real-time execution monitor (use Live Portfolio Activity for that).
- Not a run-health dashboard (use Runs for that).
- Not a standalone decision source (pair with AI Agent Decisions for full context).

## Recommended review cadence

- **Daily:** quick scan of newest chains with surprising outcomes.
- **Weekly:** cluster recurring reason/evidence patterns.
- **Monthly:** identify policy thresholds that should be tightened or relaxed.
