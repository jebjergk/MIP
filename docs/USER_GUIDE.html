1) What MIP Actually Does

MIP is an evidence-first trading system. It does not guess exact future prices. It checks what happened historically after repeatable patterns and acts only when evidence is strong.

Simple analogy

Imagine a football scout: watch matches, log repeated behavior, review outcomes, trust proven patterns, then recommend plays.

3) What's Newer In MIP

  • News Intelligence is policy-aware context: it can adjust confidence/ranking and block entries in specific conditions, but does not bypass core trust/risk gates.
  • Live-linked workflow has explicit lifecycle states: a proposal can pass committee review and still be blocked later by validation/freshness/approval constraints.
  • Decision reasoning is more structured: reason codes and revalidation outcomes are first-class diagnostics.
  • Learning Ledger closes the loop: decision-to-outcome chains are traceable for pattern review and policy tuning.
  • Runs include expected skip behavior: SUCCESS_WITH_SKIPS can be normal during no-data windows.

5) How To Read MIP Metrics

  • H1/H3/H5... represent average outcome after those forward windows.
  • Maturity is evidence confidence, not certainty.
  • Gate tells whether new entries are allowed now.
Example: Avg H5 = +0.008 means roughly +0.8% average after 5 periods.
  • APPROVED is not always EXECUTED; downstream checks can still block actions.
  • Recurring reason-code clusters usually indicate policy/freshness thresholds, not random events.

6) Page-by-Page Guide

  • Home: pre-flight checks and shortcuts.
  • Cockpit: what changed, what matters, what to watch.
  • Training Status: evidence depth by symbol/pattern.
  • Performance Dashboard: cross-portfolio return and drawdown comparison.
  • Live Symbol Tracker: symbol-first monitoring and attention routing.
  • Market Timeline: signal-to-proposal-to-trade trail.
  • Runs: operational truth source and diagnostics.
  • AI Agent Decisions: committee verdicts, reason codes, and revalidation outcomes.
  • Live Portfolio Link: live-linked controls and activation guard readiness.
  • Live Portfolio Activity: lifecycle status feed with block/delay reason fields.
  • Learning Ledger: decision-to-outcome causality and recurring motifs.
  • News Intelligence: evidence-backed news impact context and HOT indicators.

7) Daily Operating Routine

  1. Home: confirm latest successful run.
  2. Cockpit: read overnight changes.
  3. Performance Dashboard: review return and drawdown changes.
  4. Training Status: validate evidence quality.
  5. AI Agent Decisions: inspect accept/reject rationale.
  6. Live Portfolio Activity: verify expected lifecycle transitions.
  7. Runs: verify details if anything seems wrong.

8) Ask MIP Assistant

Ask MIP is route-aware help. For better answers, include portfolio/symbol/time scope and status/reason fields when diagnosing specific rows.

  • "Explain this page in plain English."
  • "What does this metric mean and what is a healthy value?"
  • "Why are proposals present but no trades executed?"

9) Practical Examples

No trades today

  1. Runs: confirm success/freshness.
  2. Training: check maturity + coverage.
  3. Portfolio: check gate/capacity.
  4. AI Agent Decisions: inspect rejections.

Metric seems wrong

  1. Confirm episode scope in Portfolio.
  2. Compare run timestamps/as-of dates.
  3. Check for recent profile/cash lifecycle events.

Approved but not executed

  1. In AI Agent Decisions, confirm latest status + reason codes.
  2. In Live Portfolio Activity, check downstream validation/compliance status.
  3. In Runs, verify whether freshness or pipeline skips explain the block.
  4. In Live Portfolio Link, confirm activation guard prerequisites.
